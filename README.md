# Домашнее задание к занятию «Репликация и масштабирование. Часть 2»

### Задание 1

Опишите основные преимущества использования масштабирования методами:

- активный master-сервер и пассивный репликационный slave-сервер; 
- master-сервер и несколько slave-серверов;


*Дайте ответ в свободной форме.*

---
### Решение 1

#### Преимущества масштабирования методом активный master-сервер и пассивный slave-сервер:

1. Отказоустойчивость: Когда master-сервер выйдет из строя, slave-сервер можно перевести в режим master, снижая время недоступности сервиса.
2. Тестирование и аналитика: Slave-сервер можно использовать для тестирования обновлений БД или выполнения аналитических запросов без нагрузки на master-сервер.
3. Резервное копирование: На slave-сервере можно выполнять бэкапы БД, без влияния на производительность master-сервера.
4. Распределение нагрузки чтения: Запросы на чтение можно перенаправлять на slave-сервер, снижая нагрузку на master-сервер.

#### Преимущества масштабирования методом master-сервер и несколько slave-серверов:
1. Отказоустойчивость: При выходе из строя одного slave-сервера остальные продолжают работать.
2. Разделение задач: Разные slave-серверы можно использовать для разных целей (тестирование, отчёты, запросы, бэкапы).
3. Масштабируемость чтения: Запросы на чтение распределяются между несколькими slave-серверами, что позволяет обслуживать большее количество пользователей.
4. Геораспределение: Slave-серверы можно размещать в разных зонах доступности для снижения задержки.

### Задание 2

Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц: 

- пользователи, 
- книги, 
- магазины (столбцы произвольно). 

Опишите принципы построения системы и их разграничение или разбивку между базами данных.

*Пришлите блоксхему, где и что будет располагаться. Опишите, в каких режимах будут работать сервера.* 

### Решение 2

Представим, что наша база состоит из трех таблиц со следующими столбцами:

1. Пользователи (*users*) - столбцы id, name, email, passwd_hash, reg_date;
2. Книги (*books*) - столбцы id, name, category_id, store_id, price;
3. Магазины (*stores*) - столбцы id, name, region, adress;

Вертикальное шардирование:

Разделим таблицу *users* на две таблицы *users_profile* и *users_detail*
Состав таблиц по столбцам:
*users_profile* - id, name, email;
*users_detail* - id, passwd_hash, reg_date;

Разделим таблицу *books* на две таблицы *books_cat* и *books_location*
Состав таблиц по столбцам:
*books_cat* - id, name, category_id;
*books_location* - id, store_id, price;

Разделим таблицу *stores* на две таблицы *stores_name* и *stores_location*
Состав таблиц по столбцам:
*stores_name* - id, name;
*stores_location* - id, city, adress;

Таблицу *users* разделим по id по определенному диапазону, например в первом шарде будут пользователи с id от 1 до 500 000, во втором шарде - от 500 001 до 1 000 000;
